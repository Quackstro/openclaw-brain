# Brain 2.0 — System Design Session

**Date:** 2026-02-01
**Status:** Design document drafted, awaiting Dr. Castro review

## Summary
Dr. Castro outlined a comprehensive vision for "AI Brain 2.0" — a behavioral support system that extends biological memory. Not a knowledge base, but a reliability engine.

## Key Decisions
- System name: Brain 2.0
- Design doc: `plans/brain-2.0-system-design.md`
- Implementation: OpenClaw plugin (`~/.openclaw/extensions/brain/`)
- Storage: LanceDB (separate from doc-RAG, same embedding infra)
- Classification: Sonnet via `llm-task` inside Lobster pipelines (96% token savings vs Opus orchestration)
- 6 buckets: People, Projects, Ideas, Admin, Documents + NeedsReview + AuditTrail
- Commands: `/drop` (capture) + `/fix` (correction)
- Digests: 4 daily + 1 weekly via cron, word-limited
- Lobster pipelines: `brain-classify.lobster`, `brain-digest.lobster`, `brain-fix.lobster`
- Confidence bouncer: ≥0.70 routes to bucket, <0.70 → needs-review

## Core Principle
User's only reliable behavior: drop thoughts when they occur. System handles everything else.

## Token Budget
~18,500 tokens/day (~$15/month) vs ~50K+/day without Lobster optimization.
